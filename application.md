# アプリケーション概要

## 設計概要
```
ユーザーがファイナンシャルプランナー(以下FP)との相談を予約できるシステム
・ユーザーおよびFPが登録出来る
  - deviseを使用したパスワード認証を実装

・予約は1枠30分
・平日の予約は10:00 ~ 18:30までの毎時0,30分からスタート
・土曜日の予約は11:00 ~ 15:00
・日曜日の予約は出来ない
  - ユーザは予約画面からFPに予約が出来る
  - 予約はReservationテーブルで管理する
  - 予約開始時間(start_at)・予約終了時間（end_at）・予約日時(reservation_date)をそれぞれ分けて保存
  - 既に予約されている時間帯は予約できない
  - FPがNG登録した時間枠は登録できない
  - 現在より過去の日時は予約できない
  - 日曜日は予約できない

・FPが予約を受け付ける枠を選択できる
  - FpNgTimeFrameテーブルで管理する。
  - FPは予約できない時間帯を選択し、保存することによってその時間帯に予約できなくすることができる。
   → 予約を受けつける枠を選択する様な実装をすると、ユーザに負担がかかる（=受け付ける枠の方が多そう）と考え予約を受け付けない枠を選択することによって受け付ける枠を選択しなくていいような実装にした。


***************************************************************************************
以下、追加機能

・ユーザは登録されたFPの一覧を参照することが出来る
  - ユーザは登録されたFPをFP一覧画面で参照し、その画面から予約画面に遷移できる
    → ユーザはFP同士を比べたい為、このページは必須だと考えた。このページに検索の機能を追加できればより使いやすくなる。

・FP、ユーザはプロフィールを編集することが出来る。
  - FPのプロフィールはFP一覧画面に反映される。
    → FPはユーザに選んでもらうためにPRは必要であり、それを登録する画面は必要だと考えた。

・FP、ユーザは予約をした・された内容を参照することが出来る
  - FP、ユーザは予約一覧画面にて、各予約内容を参照する事ができる。
    → ユーザは忘れやすいため、予約一覧を参照できるようなページは必要。

```


## 画面
```
・TOP画面
  - ユーザが最初に遷移する画面
  - ヘッダーから新規登録・ログイン・TOPへ遷移できる

・新規ユーザ登録画面
  - ユーザ登録ができる
  -「FPの方はこちら」のリンク先からFPの登録ができる

・ログイン画面
  - ユーザ・FP共にログインができる

・FP一覧画面
  - 登録されたFPの一覧を見ることが出来る
  - FPは参照することができない

・予約画面
  - 登録されたFPへの予約をすることが出来る
  - FPは参照することができない
  - 日付を選択することによって予約出来る時間帯が自動でselect box内に表示される

・予約一覧画面
  - 予約をした（された）一覧を見ることが出来る

・ダッシュボード画面（マイページ）
  - プロフィール編集、予約一覧、予約不可時間設定（FPのみ）のリンクが置かれているページ

・プロフィール編集画面
  - プロフィールを編集出来る。FPの編集された情報はFP一覧画面に反映される
```

## アピールポイント、工夫した点
```
・FPが予約を受け付ける枠を選択できる → 予約を受け付けない枠を登録することによって予約を受け付ける枠を選択しなくていいようにした点（仕様と違うのは理解しましたが、ユーザ視点で実装した）
・予約画面にて日にちを選択すると自動で予約可能な時間帯をajaxで取得し、select box内に表示させる部分
```
